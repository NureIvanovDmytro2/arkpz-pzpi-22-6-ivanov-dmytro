 Розробка бізнес-логіки та функцій адміністрування серверної частини програмної системи

Харків 2025
У лабораторній роботі було розроблено бізнес-логіку та функції адміністрування для серверної частини програмної системи, що реалізує управління транспортними засобами. Основними завданнями було створення та впровадження функціоналу для управління даними транспортних засобів (додавання, пошук, редагування та видалення) та забезпечення адміністрування системи через API.
Розробка бізнес-логіки Бізнес-логіка серверної частини була побудована з урахуванням операцій з даними транспортних засобів, таких як:
o Отримання загальної кількості транспортних засобів.
o Фільтрація та пошук транспортних засобів за роком випуску. Для цього використовувався стандартний підхід до роботи з даними через Entity Framework Core. Всі операції здійснюються на стороні сервера за допомогою методів, що взаємодіють з базою даних SQLite.
Розробка функцій адміністрування Функції адміністрування включають створення RESTful API для доступу до даних транспортних засобів. Це дозволяє адміністраторам управляти транспортними засобами, отримувати статистику, додавати нові записи або оновлювати існуючі дані. API включає методи для:
o Отримання списку транспортних засобів за певним критерієм (наприклад, за роком випуску).
o Пошук загальної кількості транспортних засобів в системі.
o Оновлення даних та видалення записів, що більше не потрібні.
